# Frontend — Labster

## Описание проекта

**Labster** — фронтенд-часть веб-приложения в формате «Tinder для домашних работ». Приложение предназначено для студентов и позволяет искать, создавать и взаимодействовать с карточками учебных заданий, находя совпадения по интересам и предметам.

Frontend отвечает за пользовательский интерфейс, клиентскую логику, маршрутизацию и взаимодействие с backend API.

## Основной функционал

### Аутентификация и регистрация

* Регистрация пользователя с указанием имени, email, города и учебного заведения
* Возможность выбора из предустановленных списков или ввода пользовательских значений
* Авторизация пользователя
* Хранение данных пользователя в `localStorage`

### Главная страница (Feed)

* Отображение списка карточек заданий
* Просмотр карточек в виде ленты
* Открытие карточки в модальном окне
* Лайк и дизлайк карточек
* Обновление списка карточек без перезагрузки страницы

### Карточки заданий

* Создание карточек заданий (предмет, тип работы, описание, курс, город, учебное заведение, файлы)
* Редактирование существующих карточек
* Удаление карточек
* Просмотр собственных карточек

### Мои карточки

* Отдельный раздел для управления карточками пользователя
* Режимы: просмотр, создание и редактирование

### Настройки пользователя

* Редактирование профиля (имя, город, учебное заведение, курс, контакты)
* Возможность смены пароля (UI-логика)

### Обработка ошибок

* Страница 404 (Not Found)
* Отображение ошибок авторизации и сетевых запросов

## Архитектура и технологии

* **Framework:** React 18
* **Язык:** TypeScript
* **Маршрутизация:** React Router v6
* **Сборка:** Webpack
* **UI:** кастомные компоненты
* **Хранение состояния:** React hooks, localStorage
* **Взаимодействие с API:** Fetch API
* **Node.js:** 18+

## Взаимодействие с backend

Frontend взаимодействует с backend через REST API:

* `/api/users` — регистрация и авторизация
* `/api/cards` — работа с карточками заданий
* `/api/like` — лайки и дизлайки карточек

Для авторизации используется идентификатор пользователя, сохранённый в `localStorage`.

## Запуск проекта локально

```bash
npm install
npm start
```

Приложение будет доступно по адресу: `http://localhost:3000`

Для сборки production-версии:

```bash
npm run build
```

## CI / GitHub Actions

В репозитории настроен **Frontend CI workflow**, который автоматически:

* запускается при push в ветку `main` или вручную
* устанавливает зависимости
* проверяет корректность сборки проекта

Файл workflow: `.github/workflows/frontend-ci.yml`

## Назначение репозитория

Данный репозиторий содержит **клиентскую часть** проекта Labster и предназначен для:

* разработки пользовательского интерфейса
* интеграции с backend
* тестирования пользовательских сценариев
